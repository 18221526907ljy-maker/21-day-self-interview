# MindFit x Odoo 工作流

这个工作流用于把 `21 天自我采访` 放进 MindFit 的 Odoo 教育版项目管理场景中。

## 业务定位

在 MindFit 中，`21 天自我采访` 不是独立医疗服务，而是一个轻量的自我反思与情绪支持流程。它适合放在：

- MindBot 轻心理支持
- 会员成长计划
- 身材管理前的动机澄清
- 社区打卡活动
- 课程项目演示

## Odoo 对象映射

| MindFit 概念 | Odoo 中的实现 |
|---|---|
| 用户 | 联系人 / 客户 |
| 21 天计划 | 项目 |
| 每日采访 | 项目任务 |
| 当天回答摘要 | 任务评论或记录 |
| MindBot 跟进 | 活动或任务负责人 |
| 需要人工关注 | 任务阶段和标签 |
| 复盘日 | Day 7、Day 14、Day 21 任务 |
| 课程展示资料 | 知识库 / 文档 / 项目描述 |

## 推荐项目设置

项目名称：

`MindFit - 21天自我采访`

项目描述：

```text
这是 MindFit 的轻心理支持流程。用户连续 21 天完成每日 3 个自我采访问题，MindBot 负责提问、记录和阶段性回看。Odoo 用于管理用户进度、每日任务、跟进状态和需要人工关注的情况。

GitHub 仓库：
https://github.com/18221526907ljy-maker/21-day-self-interview
```

## 推荐任务阶段

1. `未开始`
2. `今日进行中`
3. `已完成`
4. `需要 MindBot 跟进`
5. `需要人工关注`

## 推荐演示流程

1. 在 Odoo 创建 `MindFit - 21天自我采访` 项目。
2. 按 [`21-day-tasks.md`](21-day-tasks.md) 建立 21 个任务。
3. 创建一个示例用户联系人。
4. 把 Day 1 任务分配给该用户或项目成员。
5. 在任务评论区写入一段示例回答摘要。
6. 把 Day 7 设置为复盘任务，展示 MindBot 如何引用前 6 天内容进行回看。
7. 如果需要展示风险边界，把某个任务打上 `需关注` 标签并移动到 `需要人工关注`。

## 与 GitHub 的关系

GitHub 保存：

- skill 代码
- 问题库
- 安全边界说明
- Odoo 接入说明
- 后续开发记录

Odoo 管理：

- 用户服务流程
- 每日任务状态
- 团队协作
- 跟进责任人
- 课程或项目展示

这种方式适合 Odoo 教育版，因为它不依赖自定义模块，也不需要服务器部署。
